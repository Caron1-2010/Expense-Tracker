## Описание программы

"Expense Tracker" – это настольное графическое приложение, разработанное с использованием Python и библиотеки Tkinter, предназначенное для эффективного отслеживания и управления личными расходами. Приложение позволяет пользователям удобно добавлять новые расходы, указывая их сумму, категорию и дату.

Ключевые особенности включают:

*   **Добавление расходов:** Интуитивно понятный интерфейс для ввода новых транзакций.
*   **Табличное представление:** Отображение всех расходов в удобной для просмотра таблице.
*   **Подсчет общей суммы:** Автоматический расчет общей суммы расходов.
*   **Фильтрация:** Возможность фильтровать расходы по заданному диапазону дат и по категории.
*   **Сброс фильтра:** Легкое восстановление полного списка расходов.
*   **Сохранение и загрузка данных:** Данные автоматически сохраняются в файл `data.json` при добавлении или изменении расходов и загружаются при запуске приложения. Поддерживается также ручная загрузка/сохранение через меню.
*   **Валидация ввода:** Проверяется корректность введенных данных (положительная сумма, правильный формат даты).
*   **Интеграция с Git:** Приложение автоматически создает коммиты в локальном Git-репозитории при сохранении данных, позволяя отслеживать историю изменений.

## Как использовать

1.  **Установка:**
    *   Убедитесь, что у вас установлен Python (версия 3.6 или выше).
    *   Скачайте или клонируйте исходный код приложения (файл `main.py`).
    *   (Опционально) Инициализируйте Git-репозиторий в папке с проектом, если это не было сделано разработчиком:
        ```bash
        git init
        ```

2.  **Запуск:**
    *   Откройте терминал или командную строку.
    *   Перейдите в папку, где сохранен файл `main.py`.
    *   Запустите приложение командой:
        ```bash
        python main.py
        ```

3.  **Интерфейс:**
    *   **Добавление расхода:** Введите сумму, категорию и дату (в формате `YYYY-MM-DD`) в соответствующие поля в верхней части окна и нажмите кнопку "Добавить расход".
    *   **Просмотр расходов:** Все добавленные расходы отображаются в таблице ниже полей ввода.
    *   **Фильтрация:** Используйте поля "Начальная дата", "Конечная дата" и "Категория" для фильтрации списка. Нажмите "Фильтровать" для применения. Чтобы снять фильтры, нажмите "Сбросить фильтр".
    *   **Общая сумма:** Общая сумма отображается под таблицей. При фильтрации она пересчитывается для отфильтрованных данных.
    *   **Меню "Файл":**
        *   **Загрузить:** Открывает диалоговое окно для выбора файла (`.json`) с данными расходов.
        *   **Сохранить как:** Открывает диалоговое окно для сохранения текущих данных в файл `.json`.
        *   **Выход:** Закрывает приложение.

4.  **Сохранение данных:**
    *   Данные автоматически сохраняются в файл `data.json` в той же папке, где находится `main.py`, после каждого добавления или изменения расхода.
    *   Автоматические коммиты в Git создаются при каждом сохранении, если Git инициализирован в директории проекта.

## Примеры использования и тесты

### 1. Добвление нового расхода

*   **Действия:**
    1.  Запустите приложение.
    2.  В поле "Сумма" введите `150.50`.
    3.  В поле "Категория" введите `Еда`.
    4.  В поле "Дата" введите `2026-04-28`.
    5.  Нажмите кнопку "Добавить расход".
*   **Ожидаемый результат:**
    *   В таблице появится новая строка с данными: `150.50 | Еда | 2026-04-28`.
    *   Общая сумма увеличится на `150.50`.
    *   В консоли (если активирован вывод Git) появится сообщение о создании коммита.
    *   В файле `data.json` появится новая запись.

### 2. Фильтрация по категории

*   **Действия:**
    1.  Добавьте несколько расходов с разными категориями (например, `Еда`, `Транспорт`, `Развлечения`).
    2.  В поле "Категория" фильтра введите `Еда`.
    3.  Нажмите кнопку "Фильтровать".
*   **Ожидаемый результат:**
    *   В таблице останутся только расходы с категорией "Еда".
    *   Под таблицей отобразится сумма только для отфильтрованных расходов.

### 3. Фильтрация по диапазону дат

*   **Действия:**
    1.  Добавьте расходы за разные даты.
    2.  В поле "Начальная дата" фильтра введите `2026-04-20`.
    3.  В поля "Конечная дата" фильтра введите `2026-04-25`.
    4.  Нажмите кнопку "Фильтровать".
*   **Ожидаемый результат:**
    *   В таблице отобразятся расходы, даты которых находятся в диапазоне с 20 апреля 2026 г. по 25 апреля 2026 г. включительно.

### 4. Проверка валидации ввода

*   **Действия:**
    1.  Попытайтесь добавить расход с некорректной суммой (например, `-100` или `abc`).
    2.  Попытайтесь добавить расход с некорректной датой (например, `28-04-2026` или `2026/04/28`).
*   **Ожидаемый результат:**
    *   Появится всплывающее окно с сообщением об ошибке, и расход не будет добавлен.

### 5. Сохранение и загрузка данных

*   **Действия:**
    1.  Добавьте несколько расходов.
    2.  Закройте приложение.
    3.  Удалите файл `data.json` (для проверки загрузки с нуля) или оставьте его.
    4.  Запустите приложение.
    5.  Через меню "Файл" -> "Загрузить" выберите ранее сохраненный файл `data.json`.
    6.  Через меню "Файл" -> "Сохранить как" сохраните данные под новым именем.
*   **Ожидаемый результат:**
    *   При запуске приложение загрузит данные из `data.json` (если он существует).
    *   После ручной загрузки таблица обновится данными из выбранного файла.
    *   Новый файл будет создан с указанным именем и будет содержать текущие данные.

### 6. Проверка истории Git (если Git был инициализирован)

*   **Действия:**
    1.  Добавьте несколько расходов, вызывая автосохранение.
    2.  Откройте терминал в папке проекта.
    3.  Выполните команду `git log`.
*   **Ожидаемый результат:**
    *   Вы увидите список коммитов, созданных автоматически при каждом сохранении, с сообщениями типа "Автосохранение".
